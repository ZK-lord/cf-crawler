# Codeforces Crawler 答辩介绍稿

---

## 一、项目概述（30秒）

各位老师好，我做的是一个 **Codeforces 竞赛数据可视化分析工具**。

简单来说就是：输入一个 CF 用户名，自动抓取他所有比赛和提交记录，分析他的表现，生成一系列可视化图表报告。后端用 C 语言写，数据可视化用 ECharts。

---

## 二、核心问题：这个工具解决了什么？（20秒）

CF 官方虽然提供个人数据页面，但有三个不足：

1. **看不出补题情况** — 赛后你有没有补做比赛里没过的题，官方不展示
2. **没有难度分布统计** — 你做过的题在各个难度级别上是怎么分布的
3. **不能同时看多个人** — 想对比自己和队友的表现，只能来回切页面

我这个工具就是来解决这三个问题的。

---

## 三、系统架构（1分钟）

整体是一个**数据管线**，分五步：

```
sample_users.txt（用户名单）
    ↓
Codeforces API（HTTP 请求）
    ↓
本地缓存（data/*.json）
    ↓
核心分析算法（C 语言）
    ↓
HTML 可视化报告（ECharts）
```

技术栈：
- **HTTP 请求**：libcurl 库
- **JSON 解析**：cJSON 轻量库
- **数据分析**：C 语言实现，核心算法约 400 行
- **前端可视化**：ECharts + HTML 模板
- **交互层**：Python Web 服务器

---

## 四、核心算法：赛时/补题判定（重点讲，1分钟）

这是整个项目最有技术含量的部分。

**问题**：给你一场比赛的所有提交记录，怎么区分每道题是赛时做出来的，还是赛后补的？

**关键**：CF API 返回的每条提交都有两个时间：
- `creationTimeSeconds` — 这条提交的时间戳
- 比赛有 `startTimeSeconds` + `durationSeconds` — 可以算出比赛结束时间

**判定规则**：

```
提交时间 <= 比赛结束时间 → 赛内提交
提交时间 >  比赛结束时间 → 赛后提交
```

结合 verdict：
- 赛内提交 + AC → **赛时通过**（绿色）
- 赛内提交 + WA → **赛时未通过**（红色）
- 赛后提交 + AC → **补题通过**（黄色，独立展示在 upsolved 区域）

**时间复杂度**：遍历全部提交 × 遍历比赛，但由于数据量在万级别，秒级完成。

---

## 五、难度分布直方图（30秒）

另一个核心功能：**AC 题目难度分布直方图**。

- 统计用户所有 AC 提交，按题目 rating 分组计数
- 支持四个时间窗口切换：全部 / 近一年 / 180天 / 近一月
- 用 ECharts 柱状图展示，横轴是难度分（800-3500），纵轴是 AC 数量

**坑**：CF 的 `user.status` 接口返回的 problem 对象里有 rating 也有 points，但有些题目没有 rating，需要兼容处理。还有个 cJSON 库的 bug — `valueint` 自增后 `valuedouble` 没同步更新，导致序列化丢数据，调试了一下午才找到。

## 六、代码结构（30秒）

```
src/
  main.c           程序入口，流程编排
  cf_api.c         调用 CF API，5 个端点封装
  analyzer.c       核心分析算法（赛时/补题判定 + 难度统计）
  htmlgen.c        生成 HTML 报告
  cJSON.c          JSON 解析库（第三方）
  json_utils.c     JSON 读写工具
  rate_limiter.c   API 限速（2秒间隔防封IP）

server.py          Python Web 服务器（支持浏览器端添加用户）
web/template.html  ECharts 前端模板
```

总共约 2000 行 C 代码 + 100 行 Python。

---

## 七、演示流程（1分钟）

1. 双击 `deploy/运行.bat`，程序自动抓取数据
2. 打开 `output/index.html`，看到多用户首页
3. 点进某个用户，展示三个核心图表：
   - **Rating 变化曲线** — 折线图，可以看到他上分历程
   - **比赛题目状态** — 每场比赛每一题的绿/红/黄标记，一眼看出补题
   - **AC 难度分布直方图** — 柱状图，切换时间窗口看活跃度变化
4. 或启动 Web 服务器：F5 运行 `server.py`，浏览器输入新用户名，点"添加"

---

## 八、总结（15秒）

纯 C 语言实现了从 API 抓取、数据清洗、核心分析到报告生成的全流程。核心创新点是**赛时/补题的精确判定算法**和**多时间窗口的难度分布统计**。工具可以直接使用，也方便扩展。
